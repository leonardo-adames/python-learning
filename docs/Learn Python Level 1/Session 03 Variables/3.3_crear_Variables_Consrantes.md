## 3. DEFINICIÓN Y CREACIÓN CONSTANTES

### ¿QUÉ SON LAS CONSTANTES?

No. Python no tiene constantes verdaderas. Esto está documentado formalmente en el `manua de estilo PEP 8`, el documento oficial de estilo de Python.

Esto significa que Python no bloquea la modificación, pero hay una convención oficial para indicar que una variable debe tratarse como constante por el programador.

(Las constantes suelen definirse a nivel de módulo y escribirse en mayúsculas con guiones bajos.)

### Fuente verificable:

PEP 8 – Style Guide for Python Code, sección Naming Conventions

https://peps.python.org/pep-0008/

## Convención oficial: usar MAYÚSCULAS

Para declarar una constante, Python usa una convención, no una restricción:

* Se escriben en MAYÚSCULAS.

* Opcionalmente se separan con guiones bajos.

* Se ubican a nivel de módulo (fuera de funciones y clases).

* Tú como programador te comprometes a no modificarla, pero Python no lo impide.

Ejemplo demostrando que sí pueden modificarse:

```python
PI = 3.14159
PI = 20   # Python lo permite, aunque está mal visto

```

### Esto muestra que Python no obliga, solo recomienda.

¿Por qué Python usa convenciones en vez de imponer restricciones?
--
* Python prioriza la simplicidad del lenguaje.

* Los programadores son responsables de mantener buenas prácticas.

* Las constantes verdaderas complican la implementación dinámica de Python.

Ejemplo completo recomendado (estilo correcto)

Archivo config.py:

```python
API_KEY = "123ABC"
TIMEOUT = 30
DEBUG_MODE = False

```

Archivo main.py:

```python
from config import TIMEOUT, DEBUG_MODE

print(TIMEOUT)

```

Esto sigue exactamente la convención del PEP 8.

### ¿Existen formas de simular constantes reales?

Sí, pero no son recomendadas para casos simples.

Ejemplos:

1. Usar una clase con propiedades de solo lectura

```python
class Constantes:
    @property
    def PI(self):
        return 3.14159

C = Constantes()
print(C.PI)

```

Intentar cambiar C.PI = 10 produce error.

2. Usar typing.Final (Python 3.8+)

Fuente: Documentado en PEP 591

```python
from typing import Final

PI: Final = 3.14159

#No lo bloquea en tiempo de ejecución,
#pero herramientas como mypy muestran error si intentas modificarlo.
```

## Resumen para estudiar (verificable)

* Python no tiene constantes reales → confirmado por PEP 8.

* La convención es escribirlas en mayúsculas.

* Se colocan a nivel de módulo.

* Python sí permite modificarlas, aunque va contra la convención.

* Para mayor seguridad existen typing.Final y clases con propiedades de solo lectura.
