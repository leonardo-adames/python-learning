## 12.3 `Raise Except`

###  Lanzamiento De exepciones con `raise`

Puedes forzar que ocurra una excepción cuando se cumple una condición usando la palabra clave `raise`, seguida del tipo de excepción.<br>
Es útil para notificar errores que tú mismo detectas en tu lógica. 

### Sintaxis

```python
raise TipoDeExcepcion("Mensaje de error")
```

### Ejemplo

```python
if edad < 18:
    raise ValueError("La edad debe ser mayor o igual a 18 para realizar esta acción.")
```
Hasta ahora, los mensajes de error se han mencionado poco, pero si has probado los ejemplos,<br>
probablemente hayas visto algunos. Hay (al menos) dos tipos de errores diferenciables: errores de sintaxis y excepciones.

El propósito real de las excepciones es manejar errores de manera elegante y permitir que el programa continúe ejecutándose.

## Casos donde usar y no usar `raie`

En esta parte vamos a hacer mención de los casos comunes donde aplicar y no aplicar el lanzamiento de Excepciones con `raise`

### Casos donde usar `raise`

- Cuando se detecta un error en la lógica del programa.

- Cuando se necesita notificar un error que tú mismo detectas.

- Cuando se quiere forzar una condición específica.

### Casos donde no usar `raise`

- Cuando el error es causado por el usuario (por ejemplo, entrada incorrecta).

- Cuando el error es causado por un recurso externo (por ejemplo, archivo no encontrado).

- Cuando el error es causado por un problema del sistema (por ejemplo, memoria insuficiente).
